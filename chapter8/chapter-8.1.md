# บทที่ 8.1 ภาพรวมของระบบปฏิบัติการ

## วัตถุประสงค์และหน้าที่ของระบบปฏิบัติการ

ระบบปฏิบัติการ (Operating System: OS) คือโปรแกรมที่ทำหน้าที่ควบคุมการทำงานของโปรแกรมประยุกต์ และทำหน้าที่เป็นตัวกลางระหว่างโปรแกรมประยุกต์กับฮาร์ดแวร์คอมพิวเตอร์ โดยระบบปฏิบัติการมีวัตถุประสงค์หลักสองประการ ได้แก่

- **ความสะดวกในการใช้งาน (Convenience):** ระบบปฏิบัติการทำให้การใช้งานคอมพิวเตอร์มีความสะดวกยิ่งขึ้น
- **ประสิทธิภาพในการใช้งาน (Efficiency):** ระบบปฏิบัติการช่วยให้การใช้ทรัพยากรของระบบคอมพิวเตอร์เป็นไปอย่างมีประสิทธิภาพสูงสุด

### ระบบปฏิบัติการในฐานะตัวกลางระหว่างผู้ใช้และคอมพิวเตอร์

ฮาร์ดแวร์และซอฟต์แวร์ที่ใช้ในการให้บริการโปรแกรมประยุกต์แก่ผู้ใช้สามารถมองได้ในรูปแบบลำดับชั้น (Layered) ดังนี้

- **ผู้ใช้ปลายทาง (End User):** ไม่จำเป็นต้องทราบโครงสร้างภายในของคอมพิวเตอร์ แต่จะมองระบบคอมพิวเตอร์ในรูปของโปรแกรมประยุกต์
- **นักพัฒนาโปรแกรม (Application Programmer):** พัฒนาโปรแกรมด้วยภาษาโปรแกรมระดับสูง

โดยระบบปฏิบัติการโดยทั่วไปให้บริการในด้านต่าง ๆ ดังนี้

1. **การสร้างโปรแกรม (Program Creation):** จัดเตรียมเครื่องมือต่าง ๆ เช่น โปรแกรมแก้ไขข้อความ (Editor) และโปรแกรมตรวจสอบข้อผิดพลาด (Debugger) เพื่อช่วยนักพัฒนาในการสร้างโปรแกรม
2. **การประมวลผลโปรแกรม (Program Execution):** โหลดคำสั่งและข้อมูลเข้าหน่วยความจำหลัก เตรียมอุปกรณ์ I/O และทรัพยากรอื่น ๆ ที่จำเป็น
3. **การเข้าถึงอุปกรณ์ I/O (Access to I/O Devices):** จัดการรายละเอียดของอุปกรณ์ I/O แต่ละชนิด เพื่อให้นักพัฒนาสามารถใช้งานในระดับการอ่านและเขียนข้อมูลได้โดยตรง
4. **การควบคุมการเข้าถึงไฟล์ (Controlled Access to Files):** จัดการและป้องกันการเข้าถึงไฟล์ในระบบที่มีผู้ใช้หลายคน
5. **การควบคุมการเข้าถึงระบบ (System Access):** ควบคุมการเข้าถึงระบบและทรัพยากรต่าง ๆ เพื่อป้องกันการใช้งานโดยไม่ได้รับอนุญาต
6. **การตรวจสอบและตอบสนองต่อข้อผิดพลาด (Error Detection and Response):** จัดการข้อผิดพลาดที่เกิดขึ้นจากทั้งฮาร์ดแวร์และซอฟต์แวร์
7. **การบัญชีการใช้งาน (Accounting):** รวบรวมสถิติการใช้งานทรัพยากรและตรวจสอบประสิทธิภาพของระบบ

### อินเทอร์เฟซสำคัญในระบบคอมพิวเตอร์

- **Instruction Set Architecture (ISA):** กำหนดชุดคำสั่งภาษาเครื่องที่คอมพิวเตอร์สามารถดำเนินการได้ ถือเป็นขอบเขตระหว่างฮาร์ดแวร์และซอฟต์แวร์
- **Application Binary Interface (ABI):** กำหนดมาตรฐานสำหรับความสามารถในการพกพาโปรแกรมในรูปแบบไบนารี
- **Application Programming Interface (API):** ให้โปรแกรมสามารถเข้าถึงทรัพยากรฮาร์ดแวร์และบริการต่าง ๆ ผ่านไลบรารีของภาษาระดับสูง

### ระบบปฏิบัติการในฐานะตัวจัดการทรัพยากร

คอมพิวเตอร์คือชุดของทรัพยากรสำหรับการเคลื่อนย้าย จัดเก็บ และประมวลผลข้อมูล ระบบปฏิบัติการมีความรับผิดชอบในการจัดการทรัพยากรเหล่านี้ โดยมีลักษณะเฉพาะสองประการ ได้แก่

1. ระบบปฏิบัติการทำงานในลักษณะเดียวกับซอฟต์แวร์คอมพิวเตอร์ทั่วไป กล่าวคือ เป็นโปรแกรมที่ถูกประมวลผลโดยหน่วยประมวลผล
2. ระบบปฏิบัติการมักจะสละการควบคุมและต้องพึ่งพาหน่วยประมวลผลเพื่อให้ได้การควบคุมกลับคืนมา

## ประเภทของระบบปฏิบัติการ

### ระบบแบบ Batch และแบบ Interactive

- **ระบบแบบโต้ตอบ (Interactive System):** ผู้ใช้สามารถโต้ตอบกับคอมพิวเตอร์โดยตรงผ่านแป้นพิมพ์หรือจอแสดงผล
- **ระบบแบบ Batch:** โปรแกรมของผู้ใช้จะถูกรวบรวมและส่งให้ผู้ดำเนินการคอมพิวเตอร์ ผลลัพธ์จะถูกพิมพ์ออกมาภายหลัง

### ระบบแบบ Multiprogramming

ระบบที่พยายามใช้งานหน่วยประมวลผลให้มีประสิทธิภาพสูงสุด โดยให้ทำงานหลายโปรแกรมพร้อมกัน โปรแกรมหลายโปรแกรมจะถูกโหลดเข้าหน่วยความจำ และหน่วยประมวลผลจะสลับการทำงานระหว่างโปรแกรมเหล่านั้นอย่างรวดเร็ว

### พัฒนาการของระบบปฏิบัติการ

1. **ระบบในยุคแรก (Early Systems):** ผู้ใช้โต้ตอบกับฮาร์ดแวร์คอมพิวเตอร์โดยตรง ไม่มีระบบปฏิบัติการ ส่งผลให้เกิดปัญหาด้านการกำหนดเวลา (Scheduling) และเวลาในการตั้งค่า (Setup Time)

2. **ระบบ Batch แบบง่าย (Simple Batch Systems):** พัฒนาขึ้นเพื่อเพิ่มประสิทธิภาพในการใช้งานหน่วยประมวลผล ผู้ใช้ส่งงานให้ผู้ดำเนินการ ซึ่งจะรวบรวมงานและป้อนเข้าสู่ระบบ Monitor จะควบคุมลำดับการทำงาน

3. **ระบบ Batch แบบ Multiprogramming (Multiprogrammed Batch Systems):** แก้ปัญหาที่หน่วยประมวลผลอยู่เฉย (Idle) เนื่องจากอุปกรณ์ I/O ทำงานช้า โดยการโหลดโปรแกรมหลายโปรแกรมเข้าหน่วยความจำพร้อมกัน

4. **ระบบแบ่งเวลา (Time-Sharing Systems):** ช่วยให้ผู้ใช้หลายคนสามารถใช้งานระบบได้พร้อมกันแบบโต้ตอบ โดยหน่วยประมวลผลจะแบ่งเวลาการทำงานให้แก่ผู้ใช้แต่ละคน
