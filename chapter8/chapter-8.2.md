# บทที่ 8.2 การจัดตารางเวลา (Scheduling)

การจัดตารางเวลาถือเป็นหัวใจสำคัญของระบบ Multiprogramming โดยทั่วไปจะมีการจัดตารางเวลาอยู่ 4 ประเภท ก่อนอื่นจำเป็นต้องทำความเข้าใจแนวคิดของ **กระบวนการ (Process)** ซึ่งหมายถึงโปรแกรมที่กำลังอยู่ระหว่างการประมวลผล

ประเภทของการจัดตารางเวลามีดังนี้

| ประเภท | ความหมาย |
|--------|----------|
| Long-Term Scheduling | การตัดสินใจเพิ่มกระบวนการเข้าสู่กลุ่มที่รอการประมวลผล |
| Medium-Term Scheduling | การตัดสินใจเพิ่มจำนวนกระบวนการที่อยู่ในหน่วยความจำหลักบางส่วนหรือทั้งหมด |
| Short-Term Scheduling | การตัดสินใจว่ากระบวนการใดที่พร้อมใช้งานจะถูกประมวลผลโดยหน่วยประมวลผลต่อไป |
| I/O Scheduling | การตัดสินใจว่าคำขอ I/O ที่รอดำเนินการของกระบวนการใดจะถูกจัดการโดยอุปกรณ์ I/O ที่ว่างอยู่ |

---

## การจัดตารางเวลาระยะยาว (Long-Term Scheduling)

ตัวจัดตารางเวลาระยะยาวทำหน้าที่กำหนดว่าโปรแกรมใดจะได้รับอนุญาตให้เข้าสู่ระบบเพื่อทำการประมวลผล ซึ่งเป็นการควบคุมระดับของ Multiprogramming (จำนวนกระบวนการในหน่วยความจำ) เมื่อได้รับการยอมรับแล้ว งาน (Job) หรือโปรแกรมของผู้ใช้จะกลายเป็นกระบวนการและถูกเพิ่มเข้าสู่คิวของตัวจัดตารางเวลาระยะสั้น

สำหรับระบบแบบ Batch ตัวจัดตารางเวลาระยะยาวจะสร้างกระบวนการจากคิวโดยพิจารณาจากเกณฑ์ต่าง ๆ เช่น ระดับความสำคัญ (Priority) เวลาที่คาดว่าจะใช้ในการประมวลผล และความต้องการด้าน I/O

---

## การจัดตารางเวลาระยะกลาง (Medium-Term Scheduling)

การจัดตารางเวลาระยะกลางเป็นส่วนหนึ่งของฟังก์ชัน Swapping โดยทั่วไปการตัดสินใจในการ Swap กระบวนการเข้ามาจะขึ้นอยู่กับความจำเป็นในการจัดการระดับของ Multiprogramming รวมถึงข้อกำหนดด้านหน่วยความจำของกระบวนการที่ถูก Swap ออกไป

---

## การจัดตารางเวลาระยะสั้น (Short-Term Scheduling)

ตัวจัดตารางเวลาระยะสั้นหรือที่เรียกว่า **ตัวกระจาย (Dispatcher)** ทำงานบ่อยครั้งและตัดสินใจในระดับละเอียดว่างานใดควรจะถูกประมวลผลต่อไป

### สถานะของกระบวนการ (Process States)

ตลอดช่วงอายุการทำงานของกระบวนการ สถานะจะเปลี่ยนแปลงอยู่เสมอ โดยมีสถานะพื้นฐานอย่างน้อย 5 สถานะ ได้แก่

1. **New:** โปรแกรมได้รับการยอมรับจากตัวจัดตารางเวลาระดับสูง แต่ยังไม่พร้อมสำหรับการประมวลผล ระบบปฏิบัติการจะเริ่มต้นกระบวนการและย้ายไปสู่สถานะ Ready
2. **Ready:** กระบวนการพร้อมสำหรับการประมวลผลและรอการเข้าถึงหน่วยประมวลผล
3. **Running:** กระบวนการกำลังถูกประมวลผลโดยหน่วยประมวลผล
4. **Waiting:** กระบวนการถูกระงับการทำงานชั่วคราวเพื่อรอทรัพยากรบางอย่าง เช่น การดำเนินการ I/O
5. **Halted:** กระบวนการสิ้นสุดการทำงานและจะถูกทำลายโดยระบบปฏิบัติการ

### Process Control Block (PCB)

สำหรับแต่ละกระบวนการในระบบ ระบบปฏิบัติการจะต้องเก็บรักษาข้อมูลที่บ่งชี้สถานะของกระบวนการไว้ใน **Process Control Block** ซึ่งโดยทั่วไปจะประกอบด้วย

- **Identifier:** ตัวระบุเฉพาะของแต่ละกระบวนการที่กำลังทำงาน
- **State:** สถานะปัจจุบันของกระบวนการ
- **Priority:** ระดับความสำคัญสัมพัทธ์
- **Program Counter:** ที่อยู่ของคำสั่งถัดไปในโปรแกรมที่จะถูกประมวลผล
- **Memory Pointers:** ตำแหน่งเริ่มต้นและสิ้นสุดของกระบวนการในหน่วยความจำ
- **Context Data:** ข้อมูลที่อยู่ในรีจิสเตอร์ขณะที่กระบวนการกำลังทำงาน
- **I/O Status Information:** รวมถึงคำขอ I/O ที่ยังค้างอยู่ และอุปกรณ์ I/O ที่ถูกกำหนดให้กระบวนการนี้
- **Accounting Information:** อาจรวมถึงเวลาของหน่วยประมวลผลและเวลานาฬิกาที่ใช้ไป

### เทคนิคการจัดตารางเวลา

ระบบปฏิบัติการจะรักษาคิวจำนวนหนึ่งเพื่อจัดการกระบวนการต่าง ๆ ดังนี้

- **Long-Term Queue:** รายการงานที่รอใช้งานระบบ
- **Short-Term Queue:** รายการกระบวนการทั้งหมดที่อยู่ในสถานะ Ready ซึ่งตัวจัดตารางเวลาระยะสั้นจะเลือกกระบวนการจากคิวนี้ โดยทั่วไปใช้อัลกอริทึม Round-Robin
- **I/O Queue:** คิวสำหรับแต่ละอุปกรณ์ I/O เพื่อจัดการกระบวนการที่รอใช้งานอุปกรณ์เดียวกัน

กระบวนการที่ถูกระงับเนื่องจากการร้องขอ I/O จะถูกวางไว้ในคิว I/O ที่เหมาะสม ในขณะที่กระบวนการที่ถูกระงับเนื่องจาก Timeout จะถูกวางกลับในสถานะ Ready และเข้าสู่ Short-Term Queue
