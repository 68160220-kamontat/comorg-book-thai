# บทที่ 8.4 การจัดการหน่วยความจำของ Intel x86

นับตั้งแต่การนำสถาปัตยกรรม 32 บิตมาใช้งาน ไมโครโปรเซสเซอร์ได้พัฒนาโครงร่างการจัดการหน่วยความจำที่มีความซับซ้อนสูง ตัวอย่างที่เป็นตัวแทนได้แก่ โครงร่างที่ใช้ในสถาปัตยกรรม Intel x86

---

## พื้นที่ที่อยู่ (Address Spaces)

สถาปัตยกรรม x86 มีฮาร์ดแวร์รองรับทั้ง Segmentation และ Paging โดยสามารถปิดใช้งานกลไกทั้งสองได้ ทำให้ผู้ใช้สามารถเลือกมุมมองหน่วยความจำได้ 4 แบบ ดังนี้

1. **Unsegmented Unpaged Memory:** ที่อยู่เสมือนเป็นเดียวกับที่อยู่ฟิสิคัล เหมาะสำหรับการประยุกต์ใช้งาน Controller ที่ต้องการความซับซ้อนต่ำและประสิทธิภาพสูง
2. **Unsegmented Paged Memory:** หน่วยความจำถูกมองเป็นพื้นที่ที่อยู่เชิงเส้นแบบเพจ การป้องกันและการจัดการหน่วยความจำจะดำเนินการผ่าน Paging
3. **Segmented Unpaged Memory:** หน่วยความจำถูกมองเป็นชุดของพื้นที่ที่อยู่แบบลอจิคัล วิธีนี้ช่วยให้สามารถป้องกันข้อมูลได้ถึงระดับไบต์เดียว
4. **Segmented Paged Memory:** ใช้ Segmentation เพื่อกำหนดพาร์ทิชันหน่วยความจำแบบลอจิคัลที่มีการควบคุมการเข้าถึง และใช้ Paging เพื่อจัดการการจัดสรรหน่วยความจำภายในพาร์ทิชัน

---

## การแบ่งเซกเมนต์ใน x86 (Segmentation)

ในระบบ x86 ที่ใช้ Segmentation แต่ละที่อยู่เสมือน (หรือ Logical Address) จะประกอบด้วย **Segment Reference ขนาด 16 บิต** และ **Offset ขนาด 32 บิต**

- ด้วย Unsegmented Memory หน่วยความจำเสมือนของผู้ใช้มีขนาด 2^32 = 4 GB
- ด้วย Segmented Memory พื้นที่หน่วยความจำเสมือนรวมทั้งหมดที่ผู้ใช้มองเห็นคือ 2^46 = 64 TB
- พื้นที่ที่อยู่ฟิสิคัลใช้ที่อยู่ขนาด 32 บิต ทำให้รองรับได้สูงสุด 4 GB

### ระดับสิทธิ์การเข้าถึง (Privilege Levels)

มีระดับสิทธิ์การเข้าถึง 4 ระดับ ตั้งแต่ระดับที่มีการป้องกันสูงสุด (ระดับ 0) ไปจนถึงระดับที่มีการป้องกันน้อยที่สุด (ระดับ 3) ดังนี้

- **ระดับ 0:** สำหรับส่วนของระบบปฏิบัติการที่ดูแลการจัดการหน่วยความจำ การป้องกัน และการควบคุมการเข้าถึง
- **ระดับ 1:** สำหรับระบบปฏิบัติการส่วนใหญ่
- **ระดับ 2:** สำหรับระบบย่อยของโปรแกรมประยุกต์เฉพาะทางที่ต้องการการป้องกัน
- **ระดับ 3:** สำหรับโปรแกรมประยุกต์ทั่วไป

### Segment Selector

Segment Selector ประกอบด้วยฟิลด์ดังนี้

- **Table Indicator (TI):** บ่งชี้ว่าควรใช้ตารางเซกเมนต์แบบ Global หรือ Local สำหรับการแปลงที่อยู่
- **Segment Number:** หมายเลขของเซกเมนต์ ใช้เป็นดัชนีในตารางเซกเมนต์
- **Requested Privilege Level (RPL):** ระดับสิทธิ์ที่ร้องขอสำหรับการเข้าถึงนี้

## การแบ่งเพจใน x86 (Paging)

กลไก Paging ของ x86 เป็นการดำเนินการค้นหาตารางสองระดับ ประกอบด้วย

1. **Page Directory:** มีรายการได้สูงสุด 1,024 รายการ แบ่งพื้นที่หน่วยความจำเชิงเส้นขนาด 4 GB ออกเป็น 1,024 กลุ่มเพจ แต่ละกลุ่มมี Page Table ของตัวเอง ขนาด 4 MB
2. **Page Table:** มีรายการได้สูงสุด 1,024 รายการ แต่ละรายการสอดคล้องกับเพจขนาด 4 KB หนึ่งเพจ

สถาปัตยกรรม x86 ยังใช้ **Translation Lookaside Buffer (TLB)** ที่สามารถรองรับรายการในตารางเพจได้ 32 รายการ

### การรองรับขนาดเพจสองขนาด

x86 รองรับขนาดเพจสองขนาด ได้แก่

- **เพจขนาด 4 KB:** ต้องมีการค้นหาตารางสองระดับ
- **เพจขนาด 4 MB:** ต้องการการค้นหาตารางเพียงระดับเดียว ช่วยลดความต้องการพื้นที่จัดเก็บสำหรับการจัดการหน่วยความจำ โดยด้วยเพจขนาด 4 MB ตารางเดียวขนาด 4 KB ก็เพียงพอสำหรับการจัดการเพจทั้งหมด

## พารามิเตอร์การจัดการหน่วยความจำ x86

| พารามิเตอร์ | คำอธิบาย |
|-------------|----------|
| Base | กำหนดที่อยู่เริ่มต้นของเซกเมนต์ภายในพื้นที่ที่อยู่เชิงเส้นขนาด 4 GB |
| Limit | กำหนดขนาดของเซกเมนต์ |
| DPL (Descriptor Privilege Level) | ระบุระดับสิทธิ์ของเซกเมนต์ |
| Granularity bit (G) | บ่งชี้ว่าฟิลด์ Limit จะถูกตีความในหน่วยไบต์หรือ 4 KB |
| Present bit (P) | บ่งชี้ว่าเพจหรือตารางเพจอยู่ในหน่วยความจำหลักหรือไม่ |
| Accessed bit (A) | ถูกตั้งเป็น 1 โดยหน่วยประมวลผลเมื่อมีการอ่านหรือเขียนไปยังเพจที่สอดคล้องกัน |
| Dirty bit (D) | ถูกตั้งเป็น 1 โดยหน่วยประมวลผลเมื่อมีการเขียนไปยังเพจที่สอดคล้องกัน |
