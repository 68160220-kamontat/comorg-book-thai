# บทที่ 8.6 คำศัพท์สำคัญและคำถามทบทวน

## คำศัพท์สำคัญ (Key Terms)

| คำศัพท์ | ความหมาย |
|---------|---------|
| Address Space | พื้นที่ที่อยู่ที่กระบวนการสามารถอ้างอิงได้ |
| Base Address | ที่อยู่เริ่มต้นของกระบวนการในหน่วยความจำ |
| Compaction | กระบวนการรวมพื้นที่ว่างในหน่วยความจำให้เป็นบล็อกเดียว |
| Demand Paging | เทคนิคที่โหลดเพจเข้าหน่วยความจำเมื่อมีการร้องขอเท่านั้น |
| Domain | กลุ่มของพื้นที่หน่วยความจำที่มีระดับการป้องกันเดียวกัน (ใน ARM) |
| Frame | ส่วนย่อยของหน่วยความจำหลักที่มีขนาดคงที่ |
| Interrupt | สัญญาณจากฮาร์ดแวร์ที่แจ้งให้หน่วยประมวลผลหยุดทำงานปัจจุบันชั่วคราว |
| Kernel | ส่วนหลักของระบบปฏิบัติการที่อยู่ในหน่วยความจำตลอดเวลา |
| Logical Address | ที่อยู่ที่แสดงเป็นตำแหน่งสัมพัทธ์จากจุดเริ่มต้นของโปรแกรม |
| Memory Management | กระบวนการแบ่งย่อยและจัดสรรหน่วยความจำสำหรับกระบวนการต่าง ๆ |
| Multiprogramming | เทคนิคที่ให้หน่วยประมวลผลทำงานกับหลายโปรแกรมพร้อมกัน |
| Page | ส่วนย่อยของโปรแกรมที่มีขนาดคงที่ |
| Page Fault | เหตุการณ์ที่เกิดขึ้นเมื่อโปรแกรมอ้างอิงเพจที่ไม่อยู่ในหน่วยความจำหลัก |
| Page Table | ตารางที่เก็บความสัมพันธ์ระหว่างเพจของกระบวนการกับเฟรมในหน่วยความจำ |
| Physical Address | ที่อยู่จริงในหน่วยความจำหลัก |
| Process | โปรแกรมที่กำลังอยู่ระหว่างการประมวลผล |
| Process Control Block (PCB) | โครงสร้างข้อมูลที่เก็บข้อมูลสถานะของกระบวนการ |
| Resident Monitor | ส่วนของ Monitor ที่อยู่ในหน่วยความจำหลักตลอดเวลา |
| Scheduling | กระบวนการตัดสินใจว่าทรัพยากรจะถูกจัดสรรให้กับกระบวนการใด |
| Segment | ส่วนย่อยของหน่วยความจำที่มีขนาดแปรผัน |
| Swapping | การย้ายกระบวนการระหว่างหน่วยความจำหลักและดิสก์ |
| Thrashing | สภาวะที่หน่วยประมวลผลเสียเวลาส่วนใหญ่ในการสลับเพจแทนการประมวลผลคำสั่ง |
| Time Sharing | เทคนิคที่แบ่งเวลาของหน่วยประมวลผลให้กับผู้ใช้หลายคนพร้อมกัน |
| TLB (Translation Lookaside Buffer) | แคชพิเศษที่เก็บรายการในตารางเพจที่ถูกใช้งานล่าสุด |
| Virtual Memory | หน่วยความจำเสมือนที่ปรากฏแก่โปรแกรม ซึ่งอาจมีขนาดใหญ่กว่าหน่วยความจำจริง |

## คำถามทบทวน (Review Questions)

1. ระบบปฏิบัติการมีวัตถุประสงค์หลักสองประการคืออะไร?

2. อธิบายความแตกต่างระหว่างที่อยู่แบบลอจิคัลและที่อยู่แบบฟิสิคัล

3. การ Multiprogramming ช่วยเพิ่มประสิทธิภาพของหน่วยประมวลผลได้อย่างไร?

4. อธิบายสถานะของกระบวนการ (Process States) ทั้ง 5 สถานะ

5. Demand Paging แตกต่างจาก Paging แบบธรรมดาอย่างไร?

6. Translation Lookaside Buffer (TLB) ทำงานอย่างไร และมีประโยชน์อย่างไร?

7. อธิบายความแตกต่างระหว่าง Paging และ Segmentation

8. ในสถาปัตยกรรม x86 การใช้เพจขนาด 4 MB แทนเพจขนาด 4 KB มีข้อดีอย่างไร?

9. ระบบ ARM ใช้กลไก Domain สำหรับการควบคุมการเข้าถึงหน่วยความจำอย่างไร?

10. อธิบายความแตกต่างระหว่าง Long-Term Scheduling, Medium-Term Scheduling และ Short-Term Scheduling

## สรุปเนื้อหาบทที่ 8

บทที่ 8 กล่าวถึงบทบาทของระบบปฏิบัติการในการสนับสนุนการทำงานของระบบคอมพิวเตอร์ โดยครอบคลุมหัวข้อสำคัญดังนี้

- **ภาพรวมของระบบปฏิบัติการ:** วัตถุประสงค์ หน้าที่ และวิวัฒนาการของระบบปฏิบัติการตั้งแต่ยุคแรกจนถึงปัจจุบัน
- **การจัดตารางเวลา:** การจัดการกระบวนการผ่านการจัดตารางเวลาสามระดับ ได้แก่ ระยะยาว ระยะกลาง และระยะสั้น รวมถึงแนวคิดของสถานะกระบวนการและ Process Control Block
- **การจัดการหน่วยความจำ:** เทคนิคต่าง ๆ ในการจัดการหน่วยความจำ ได้แก่ Swapping, Partitioning, Paging, Virtual Memory, TLB และ Segmentation
- **การจัดการหน่วยความจำของ Intel x86:** โครงร่างการจัดการหน่วยความจำของสถาปัตยกรรม x86 ที่รวม Segmentation และ Paging เข้าด้วยกัน
- **การจัดการหน่วยความจำของ ARM:** สถาปัตยกรรมระบบหน่วยความจำเสมือนของ ARM ที่มีความยืดหยุ่นสูงสำหรับระบบสมองกลฝังตัว
